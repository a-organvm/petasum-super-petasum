## 2024-05-23 - Broken YAML as a Hiding Place for Injections
**Vulnerability:** Found a GitHub Actions workflow with a broken YAML structure (top-level `env` and `run` blocks interspersed with `steps`) that `pyyaml` parses successfully but which confuses human reviewers and potentially the workflow runner. The workflow also contained multiple script injection vulnerabilities via direct `${{ ... }}` interpolation of user-controlled issue labels into `run` scripts and `actions/github-script` JavaScript.
**Learning:** Broken or confusing YAML indentation/structure can be valid YAML but functionally incorrect in GitHub Actions context. This structural chaos can mask severe security issues like script injection because reviewers might focus on "fixing the syntax" rather than seeing the injection.
**Prevention:** Always validate workflows with `action-validator` or similar tools, not just generic YAML parsers. When fixing structure, explicitly hunt for injections in the re-ordered steps. Use `env` variables for all user inputs in shell scripts and `context.payload` in JS actions.
