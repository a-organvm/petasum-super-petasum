## 2024-10-26 - Injection Vulnerabilities in GitHub Actions
**Vulnerability:** Found critical shell and JavaScript injection vulnerabilities in `org-issue-notifications.yml`. The workflow interpolated untrusted input (issue labels, outputs) directly into Bash commands and JavaScript code blocks using `${{ ... }}`. It also contained a severe YAML syntax error (orphaned `run` block) that likely caused the workflow to fail or behave unpredictably.
**Learning:** Developers often mistake `${{ ... }}` interpolation for safe variable passing. In GitHub Actions, this is a macro replacement before execution, meaning malicious input can break out of string delimiters in both Shell and JS. The YAML error highlights the lack of automated validation for workflow files in this repository.
**Prevention:** ALWAYS use environment variables (`env: ...`) to pass data to Shell `run` blocks, or access the `context.payload` object directly within `actions/github-script` steps. Never interpolate `${{ }}` inside a script. Use `actions/github-script` to consolidate logic and reduce reliance on fragile shell steps.
