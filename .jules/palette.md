## 2025-11-18 - Documentation Navigation Standardization
**Learning:** In long Markdown documentation files without a frontend framework, users lack instant navigation to the top. Adding `<a id="top"></a>` after the H1 and `[Back to Top](#top)` links before major sections (Level 2 headers) and at the end of the file significantly improves navigability.
**Action:** Apply this "Back to Top" pattern to all documentation files exceeding 3 screens in length, ensuring links are placed before section breaks for visual consistency.
